# Work Quality Patterns

1. Trial-and-error instead of reading code to understand it first
2. Making changes without verifying them (no check/test run after edits)
3. Fixing symptoms instead of root causes
4. Changing more files than necessary to solve the problem
5. Copy-pasting code instead of extracting shared logic
6. Leaving debug artifacts (console.log, commented-out code, temporary files)
7. Making an edit then immediately making another edit to the same file to fix the first edit
8. Not reading a file before editing it
9. Running a command, getting an error, and running the same command again expecting different results
10. Solving a different problem than the one that was asked about
11. Did not create a plan before starting work
